Web Components: A React Developer's Guide

Learn Web Components by comparing them to React patterns you already know

What are Web Components?

Web Components are a set of web platform APIs that allow you to create new custom, reusable, encapsulated HTML tags. Think of them as native HTML elements that you can create yourself - similar to React components but built into the browser.

React Developer Perspective:

If you're familiar with React, Web Components are like creating custom HTML elements that have their own state, props (attributes), and lifecycle methods - but they work in any framework or vanilla JavaScript!

Step 1: Basic Component (Like a Simple React Component)

React Equivalent: 

function MyButton({ title }) {
  return <button className="bg-blue-500 text-white px-4 py-2 rounded">{title}</button>;
}

Web Component:

class MyButton extends HTMLElement {
  constructor() {
    super();
    // Like React's constructor
  }

  connectedCallback() {
    // Like React's componentDidMount
    this.innerHTML = `<button class="bg-blue-500 text-white px-4 py-2 rounded">${this.getAttribute('title')}</button>`;
  }
}

customElements.define('my-button', MyButton);

Step 2: Props (Attributes) - Like React Props

React Equivalent: 

function UserCard({ name, age, email }) {
  return (
    <div className="border border-gray-300 p-4 m-2 rounded-lg bg-white">
      <h3 className="text-lg font-semibold text-gray-800">{name}</h3>
      <p className="text-gray-600">Age: {age}</p>
      <p className="text-gray-600">Email: {email}</p>
    </div>
  );
}

Web Component:

class UserCard extends HTMLElement {
  static get observedAttributes() {
    return ['name', 'age', 'email']; // Like React propTypes
  }

  connectedCallback() {
    this.render();
  }

  attributeChangedCallback(name, oldValue, newValue) {
    // Like React's componentDidUpdate
    this.render();
  }

  render() {
    const name = this.getAttribute('name') || 'Unknown';
    const age = this.getAttribute('age') || '0';
    const email = this.getAttribute('email') || 'No email';
    
    this.innerHTML = `
      <div class="border border-gray-300 p-4 m-2 rounded-lg bg-white">
        <h3 class="text-lg font-semibold text-gray-800">${name}</h3>
        <p class="text-gray-600">Age: ${age}</p>
        <p class="text-gray-600">Email: ${email}</p>
      </div>
    `;
  }
}

customElements.define('user-card', UserCard);

Step 3: State Management (Like React useState)

React Equivalent: 

function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div className="text-center p-5 bg-gray-100 rounded-lg">
      <p className="text-2xl font-bold">Count: {count}</p>
      <button onClick={() => setCount(count + 1)} className="bg-green-500 text-white px-4 py-2 rounded mr-2">+</button>
      <button onClick={() => setCount(count - 1)} className="bg-red-500 text-white px-4 py-2 rounded">-</button>
    </div>
  );
}

Web Component:

class Counter extends HTMLElement {
  constructor() {
    super();
    this.count = 0; // Like React state
  }

  connectedCallback() {
    this.render();
    this.setupEventListeners();
  }

  setupEventListeners() {
    this.addEventListener('click', (e) => {
      if (e.target.classList.contains('increment')) {
        this.count++;
        this.render(); // Like setState triggering re-render
      } else if (e.target.classList.contains('decrement')) {
        this.count--;
        this.render();
      }
    });
  }

  render() {
    this.innerHTML = `
      <div class="text-center p-5 bg-gray-100 rounded-lg">
        <p class="text-2xl font-bold">Count: ${this.count}</p>
        <button class="increment bg-green-500 text-white px-4 py-2 rounded mr-2">+</button>
        <button class="decrement bg-red-500 text-white px-4 py-2 rounded">-</button>
      </div>
    `;
  }
}

customElements.define('my-counter', Counter);

Key Differences: Web Components vs React

Web Components

  • • Native browser APIs
  • • Work in any framework
  • • Use Tailwind/CSS for styling
  • • Custom elements are real HTML elements
  • • Attributes for props
  • • Custom events for communication
  • • No virtual DOM

React

  • • JavaScript library
  • • React ecosystem only
  • • CSS-in-JS or CSS modules
  • • JSX returns virtual DOM
  • • Props as function parameters
  • • Callback functions for communication
  • • Virtual DOM for performance

When to Use Web Components

✅ Good for:

  • • Creating reusable UI components across different frameworks
  • • Building design systems that work everywhere
  • • Encapsulating complex functionality
  • • When you want to use familiar CSS frameworks like Tailwind

⚠️ Consider React instead for:

  • • Complex state management
  • • Large applications with many components
  • • When you need React's ecosystem
  • • Server-side rendering
